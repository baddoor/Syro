# 同步、缓存与 Overlay

## 这是什么
- 这一页解释自动增量同步、缓存持久化、同步进度显示和 Overlay 合并的关系。
- 它的目标不是让你天天手工操作，而是帮你在出现“看起来没更新”时知道该查哪里。

## 从哪里进入
- 自动增量同步设置、缓存持久化设置、同步进度显示设置。
- 重建缓存、全量同步、清理 ghost cards 等维护命令。

## 适合什么场景
- 你刚修改了很多笔记，却发现卡片数量或队列状态没有马上变化。
- 你想理解重启后为什么有时很快，有时需要重新扫描。
- 你在排查历史残留、ghost cards 或状态不一致问题。

## 具体步骤
1. 先确认自动增量同步和缓存持久化的当前设置。
2. 把日常情况和排障情况分开：日常主要依赖自动增量同步；排障时才考虑全量重建或更重的维护动作。
3. 如果你看到状态不一致，先给同步和 Overlay 合并一点时间，不要立刻手工删文件。
4. 当你确认自动路径没有恢复正常，再逐步升级到重建缓存、检查 ghost cards 或更深入的数据排查。

## 常见错误
- 日常每次都做重建缓存，把重操作当成普通流程。
- 同步刚开始就判断“坏了”，没有给增量流程留出时间。
- 一看到 Overlay 文件就直接删除。

## FAQ
- **自动增量同步和重建缓存谁更常用**：绝大多数情况下是自动增量同步。重建缓存更偏向排障或大规模结构变化后的恢复动作。
- **缓存持久化会不会造成旧状态残留**：它主要是性能优化；真正的异常判断仍需要结合同步、Overlay 和实际文件变化一起看。
- **进度显示模式会影响结果吗**：不会。它影响的是你能不能及时看到同步提示。

## 排错与风险提示
- 当你同时改了大量文本、设置和数据文件时，问题可能看起来像“同步异常”，实际上是多种因素叠加。
- 如果你要排查更深层的数据一致性，先备份，再进入手工检查流程。
