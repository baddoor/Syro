# 数据与同步总览

## 这是什么
- 这一组页面解释 Syro 如何存放数据、什么叫 Overlay、什么时候需要同步或重建缓存，以及旧数据模式如何迁移。
- 如果你准备备份、迁移、比较不同版本行为，或者排查“界面和实际文件对不上”的问题，这组页面必须看。

## 从哪里进入
- 日常用户通常通过同步命令、缓存重建入口和备份流程接触这一组主题。
- 更底层的数据文件说明放在本组和附录里，不需要新用户一开始全部掌握。

## 适合什么场景
- 你做了大批量文本修改，想知道是否需要同步或重建缓存。
- 你看到插件目录里出现多个 JSON 文件，想知道各自干什么。
- 你准备升级、迁移或手工备份数据。

## 具体步骤
1. 先从 [存储模型](./storage-model.md) 了解主数据、笔记复习数据、提交记录和 AI 主题包数据分别放在哪里。
2. 再读 [同步、缓存与 Overlay](./sync-cache-and-overlay.md)，理解为什么改完笔记后界面有时不会立刻同步更新。
3. 如果你来自旧版本或旧教程，继续读 [旧存储迁移](./legacy-storage-migration.md)。
4. 真正要手工操作文件前，再回到 [数据文件参考](../appendix/data-files-reference.md) 做最后确认。

## 相关设置 / 相关命令
- 相关页面： [数据文件参考](../appendix/data-files-reference.md)、[数据同步与迁移问题](../troubleshooting/data-sync-and-migration-issues.md)。
- 算法和设置变化会影响你如何理解这些数据，但不会替代这一组页面。

## 常见错误
- 只看到一个 JSON 文件就以为它是全部真相。
- 同步异常时立刻手工删文件，而没有先理解 Overlay 和缓存的作用。
- 沿用旧版本教程，把历史存储模式写成当前正式推荐方案。

## FAQ
- **为什么插件目录里会有多个数据文件**：因为卡片、笔记复习、提交记录、Overlay 和 AI 主题包并不完全属于同一类数据。
- **我需要天天手工备份所有文件吗**：不需要，但在迁移、升级、大改动或排障前，完整备份会非常有帮助。
- **为什么这里反复强调“当前正式支持的存储方式”**：因为旧模式仍存在于历史逻辑或兼容字段里，但不应该再被当成新用户主方案。

## 排错与风险提示
- 如果你不理解数据文件之间的关系就手工删除，很容易把“暂存状态”误删成“主数据损坏”。
- 升级和迁移前请先备份整个插件数据目录，而不是只复制你看得懂的那一个文件。
