# 牌组树与复习入口

![牌组树中的复习状态](../../assets/media/en/review-deck.jpg)

> 提示：当前仓库可复用的截图多来自较早的英文界面，但布局和入口位置仍可作为对照。

## 这是什么
- 牌组树是闪卡工作流的总控台。它告诉你今天有哪些卡待处理、在哪些牌组里，以及你是从整库、单牌组还是单笔记进入。
- 只要你明白牌组树，很多“为什么这里显示的数量和我想的不一样”都会更容易解释。

## 从哪里进入
- 整库复习、集中复习、当前笔记复习、当前笔记集中复习。
- 状态栏或 Ribbon 的闪卡入口。
- 从闪卡工作区进入后看到的第一个总览界面。

## 适合什么场景
- 你想快速扫一眼今天哪些牌组最值得先开。
- 你只想处理当前笔记里的卡片，而不打断其他牌组的节奏。
- 你想理解 New / Learn / Due 在 Syro 里的日常含义。

## 具体步骤
1. 先从整库入口进入牌组树，观察顶层与子牌组的层级关系。
2. 看清 New、Learn、Due 三列，它们决定的是“今天看到什么”和“以什么状态进入会话”。
3. 测试折叠与展开，确认你是否需要保持某些子牌组常开。
4. 再试一次“只复习当前笔记中的卡片”，体会它和整库入口在范围上的区别。
5. 如果某个牌组节奏不对，再通过齿轮按钮进入牌组选项，而不是直接猜测算法出错。

## 常见错误
- 把牌组树里的数量直接当成“我仓库里的全部卡片数”，忽略了同步和当前过滤范围。
- 从错误的入口进入，比如明明只想看当前笔记，却开了整库复习。
- 看到空牌组就认定卡片消失，而没有先判断当前是否真的解析出卡片。

## FAQ
- **New / Learn / Due 分别是什么意思**：可以简单理解成新卡、学习中卡、到期复习卡。具体调度细节由算法决定，但这三类足以帮助你阅读界面。
- **我能只开一个子牌组吗**：可以。你既可以从牌组树里进入，也可以先通过当前笔记入口把范围缩小。
- **为什么牌组树和队列侧边栏长得完全不一样**：因为它们分别服务于闪卡和笔记两条工作流。

## 排错与风险提示
- 如果你频繁改动牌组、标签或卡片来源，先完成同步再判断牌组树是否异常。
- 如果你准备手动清理 ghost cards，先读完 [牌组选项与同步](./deck-options-and-sync.md) 和 [数据与同步总览](../data-and-sync/index.md)。
