# 附录总览

## 这是什么
- 附录用于集中收录速查表：命令索引、快捷键索引、设置索引和数据文件参考。
- 它不是主教程，但在你需要快速定位某个名字、某个文件或某个快捷键时非常有用。

## 从哪里进入
- 当你已经理解主工作流，只想快速查某个术语或入口时。
- 当你写教程、整理发布说明或做排障笔记时。

## 适合什么场景
- 你忘了某个命令的名字。
- 你想确认某个快捷键有没有现成入口。
- 你需要把设置项和数据文件快速对照出来。

## 具体步骤
1. 找命令就看 [命令索引](./command-index.md)。
2. 找快捷键就看 [快捷键索引](./shortcut-index.md)。
3. 找设置项就看 [设置索引](./settings-index.md)。
4. 找文件说明就看 [数据文件参考](./data-files-reference.md)。

## 相关设置 / 相关命令
- 主教程入口： [中文文档总览](../index.md)。
- 排障入口： [故障排查总览](../troubleshooting/index.md)。

## 常见错误
- 把附录当成主教程从头读到尾。
- 只记住内部字段名，不记用户能看到的入口名称。
- 遇到问题只查索引，不回到主教程理解上下文。

## FAQ
- **附录是不是给开发者看的**：不只是。对重度用户、维护者和写教程的人同样有用。
- **为什么有些信息在主教程和附录里都出现了**：主教程讲上下文，附录讲速查；两者职责不同。
- **索引会替代正文吗**：不会。索引只帮你快找，不负责解释为什么。

## 排错与风险提示
- 索引类页面最容易被断章取义；真正操作前最好回到对应正文页再确认一次。
- 如果你准备对外引用这些名字，请尽量使用用户能看见的命令名和页面名。
