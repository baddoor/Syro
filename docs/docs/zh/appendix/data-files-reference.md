# 数据文件参考

## 这是什么
- 这一页用速查表方式回顾当前文档提到的数据文件，以及它们在备份、排障和迁移中的意义。

## 从哪里进入
- 当你在插件目录里看到多个 JSON 文件时。
- 当你准备做完整备份或对照数据状态时。

## 适合什么场景
- 你需要快速确认一个文件是不是主数据、辅助数据还是实验数据。
- 你想知道备份时哪些文件要一起复制。

## 具体步骤
1. 主卡片与追踪状态优先关注 `tracked_files.json`。
2. 暂存与增量层关注 `tracked_files.review_overlay.json`。
3. 笔记复习状态关注 `review_notes.json`，提交记录关注 `review_commits.json`。
4. AI 主题包相关状态关注 `ai_theme_decks.json`。

## 相关设置 / 相关命令
- 相关页面： [存储模型](../data-and-sync/storage-model.md)、[旧存储迁移](../data-and-sync/legacy-storage-migration.md)。

## 常见错误
- 把辅助数据文件看成可以随便删掉的缓存。
- 只备份自己看得懂名字的一个文件。

## FAQ
- **为什么这里不写绝对路径**：因为不同平台和安装方式下路径可能不同，文档重点放在文件职责和组合关系。
- **所有文件都需要手工编辑吗**：不建议。多数情况下你只需要理解它们，不需要直接编辑。

## 排错与风险提示
- 手工编辑任何数据文件前请先完整备份。
- 如果你不确定某个文件是否安全可删，默认先不要删，再回到对应正文确认。
