# 设置总览

![设置界面](../../assets/media/en/settings-notes.jpg)

> 提示：当前仓库可复用的截图多来自较早的英文界面，但布局和入口位置仍可作为对照。

## 这是什么
- 这一组页面负责解释当前稳定可见设置、历史兼容字段、算法参数以及界面与同步选项。
- 文档以当前代码真实行为为准；看到旧字段、旧文案或未完全接线的控件时，会在对应页面显式说明。

## 从哪里进入
- 插件设置页中的 Flashcards、Notes、Algorithm、UI、Sync 等标签。
- 与支持者能力相关的可见开关会在本组页面标记，但更完整的边界说明放在实验与高级区。

## 适合什么场景
- 你已经跑通了基本工作流，想开始做稳定配置。
- 你想弄清哪些设置是真正推荐给普通用户的，哪些只是兼容字段或预留项。
- 你需要解释某个设置为什么改了以后体感变化很大，或者为什么几乎没有变化。

## 具体步骤
1. 先看 [稳定设置与兼容字段](./stable-vs-legacy.md)，建立“哪些开关是真正值得日常关心的”这个边界。
2. 再看 [卡片与笔记设置](./flashcards-and-notes.md)，这里覆盖最常用的卡片解析、上下文、分隔符、队列和 Timeline 设置。
3. 如果你要调算法，再进入 [算法与 WMS](./algorithms-and-wms.md)。
4. 当你需要整理界面、状态栏、同步提示和缓存持久化时，看 [界面与同步](./ui-and-sync.md)。

## 相关设置 / 相关命令
- 数据文件和迁移说明在 [数据与同步总览](../data-and-sync/index.md)。
- 解析异常和设置误解的排查在 [设置与解析问题](../troubleshooting/settings-and-parse-issues.md)。

## 常见错误
- 没有先区分稳定设置和兼容字段，就开始照着旧教程一项一项抄。
- 把 UI 层面的节奏控制和算法层面的调度控制混为一谈。
- 看到一个设置项就默认它一定已经完整接入当前实现。

## FAQ
- **我应该先看哪个设置页**：大多数用户先看卡片与笔记设置，再看界面与同步；算法页通常只在你已经有明确问题时才需要。
- **所有设置都在当前 UI 里吗**：不是。代码里仍然保留了一些历史兼容字段，文档会把它们单独标出来。
- **为什么这里会反复强调“代码真实行为”**：因为旧字段、旧文案和预留控件仍然存在，如果只看名字很容易误判它们的实际作用。

## 排错与风险提示
- 在不了解作用范围前不要同时调整大量设置；先小步验证，再推广到整个库。
- 如果你需要根据设置判断数据状态，请把本组页面和 [数据与同步](../data-and-sync/index.md) 一起看。
