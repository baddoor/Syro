# Syro 的所有入口

![Syro 入口总览](../../assets/media/en/user-interface-overview.jpg)

> 提示：当前仓库可复用的截图多来自较早的英文界面，但布局和入口位置仍可作为对照。

## 这是什么
- 这一页专门解释 Syro 的稳定入口。你不需要记住所有按钮，但需要知道“哪一类入口适合什么场景”。
- 文档以当前代码里明确注册的入口为准：命令面板、状态栏、Ribbon、文件右键、文件夹右键、侧边栏和复习视图。

## 从哪里进入
- 命令面板：搜索 `Syro` 即可看到笔记复习、闪卡复习、同步、延期等命令。
- 状态栏：当前实现会提供笔记复习和闪卡复习两个快速入口。
- Ribbon：适合固定打开闪卡复习入口。
- 文件右键：适合追踪当前笔记、取消追踪、就地评分和设置重要性。
- 文件夹右键：适合批量追踪或取消追踪整个目录。
- 侧边栏 / 队列视图：适合在右侧长期停留笔记复习队列。

## 适合什么场景
- 你想从最通用的地方先确认插件有没有正常工作。
- 你在读一篇笔记时，想就地把它纳入复习。
- 你已经形成固定习惯，想把按钮放在状态栏、Ribbon 或侧边栏里。
- 你需要给某个文件夹批量建立复习上下文。

## 具体步骤
1. 优先用命令面板做第一次验证。只要你能搜到 Syro 命令，说明插件的命令注册是正常的。
2. 日常复习时优先用状态栏或 Ribbon 进入，因为这些入口最短、最不打断阅读。
3. 当你正在浏览具体笔记或目录时，用右键菜单会更高效；它允许你就地追踪、取消追踪，或直接把当前上下文送进复习流程。
4. 如果你需要持续观察“还有哪些笔记待看”，把笔记复习侧边栏固定在工作区右侧。
5. 如果你想让入口更稳定，先跑通一个主工作流，再决定哪些按钮要留在工作区。

## 常见错误
- 只在一个入口里找功能，忽略了其他入口更适合当前场景。
- 把文件右键和文件夹右键混为一谈，结果以为“批量追踪”在单文件菜单里丢失了。
- 看到状态栏按钮就直接开始复习，却没有先理解它会带你进入哪条工作流。

## FAQ
- **为什么我在不同地方看到的功能不完全一样**：因为 Syro 的入口是按场景拆开的。命令面板覆盖最全，右键菜单偏就地操作，状态栏和 Ribbon 偏快速进入。
- **右键菜单会不会修改我的原文**：追踪和评分本身主要修改插件维护的数据，而不是随意改写你的笔记正文；但与标签、frontmatter 相关的动作应先确认你当前的写作习惯。
- **如果我只保留一个入口，建议留哪个**：对多数用户来说，命令面板 + 一个常驻入口已经够用。需要长期跟踪笔记时，再加右侧队列视图。

## 排错与风险提示
- 如果当前活动文件不是 Markdown，某些“就地复习”入口不会生效。
- 如果你刚完成大量文件调整，先让同步跑完，再通过右键或状态栏进入复习，能减少“看起来没更新”的错觉。
