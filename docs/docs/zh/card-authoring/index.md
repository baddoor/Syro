# 卡片编写总览

![Cloze 卡片示例](../../assets/media/en/flashcard-cloze-example.jpg)

> 提示：当前仓库可复用的截图多来自较早的英文界面，但布局和入口位置仍可作为对照。

## 这是什么
- 这一组页面解释 Syro 如何从你的笔记里识别卡片，以及你应该怎样写得更稳定、更易维护。
- 如果你经常遇到“为什么没出卡”“为什么卡片显示很怪”，这组页面比算法页更重要。

## 从哪里进入
- 普通问答卡来自文本分隔符。
- Cloze 卡可以来自高亮、粗体、花括号、Anki 风格语法，以及支持者相关的代码块或 LaTeX 能力。
- 命令也提供创建同级 / 新级 Cloze 的快捷入口。

## 适合什么场景
- 你想把一段现有笔记稳稳地变成卡片。
- 你想决定什么时候用 Q/A，什么时候用 Cloze。
- 你想理解代码块、LaTeX、Anki Cloze 为什么属于高级场景。

## 具体步骤
1. 先读 [问答卡](./qa-cards.md)，掌握最简单、最稳定的写法。
2. 再读 [Cloze 工作流](./cloze-workflow.md)，理解高亮、粗体、花括号和快捷命令如何转换成 Cloze。
3. 最后再看 [高级 Cloze](./advanced-cloze.md)，只在你真的有代码、公式或跨段上下文需求时开启。

## 相关设置 / 相关命令
- 相关设置：卡片分隔符、Cloze 上下文模式、性能模式、显示其他 Cloze。
- 相关页面： [卡片与笔记设置](../settings/flashcards-and-notes.md)、[故障排查](../troubleshooting/settings-and-parse-issues.md)。

## 常见错误
- 一开始就使用最复杂的高级写法，结果定位问题非常困难。
- 同时混用多种语法，却没有给自己建立统一写作习惯。
- 把“卡片写法问题”误判成“算法问题”或“数据问题”。

## FAQ
- **新用户应该先学哪种写法**：先学最简单的问答卡和基础 Cloze，再逐步加高级能力。
- **Anki Cloze 能不能直接拿来用**：能否直接复用取决于当前设置和支持者能力是否启用；详细见后续页面。
- **高级 Cloze 一定更好吗**：不一定。它更强大，也更容易引入性能、可读性和维护成本。

## 排错与风险提示
- 如果你在同一库里为不同项目混用完全不同的写法，以后排错会很痛苦。
- 支持者能力、代码块解析和 LaTeX 相关设置涉及更多前置条件，开启前先确认你的真实需求。
