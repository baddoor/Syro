# Cloze 工作流：从高亮到命令快捷键

![Cloze 示例](../../assets/media/en/flashcard-cloze-example.jpg)

> 提示：当前仓库可复用的截图多来自较早的英文界面，但布局和入口位置仍可作为对照。

## 这是什么
- Cloze 工作流适合你已经习惯在原文里保留上下文，只想把关键片段挖空。
- Syro 当前支持通过高亮、粗体、花括号以及专门命令创建 Cloze。

## 从哪里进入
- 高亮、粗体、花括号等文本写法。
- 命令 `Create Cloze (Same Level)` 和 `Create Cloze (New Level)`。
- 与 Cloze 上下文、显示和性能相关的设置。

## 适合什么场景
- 你希望保留原句结构，只屏蔽最关键的词或短语。
- 你已经在日常笔记里大量使用高亮或粗体，希望顺手转成卡片。
- 你想用快捷键快速构造多空 Cloze。

## 具体步骤
1. 先选一种最符合你写作习惯的基础来源：高亮、粗体或花括号，不要一开始三种都混用。
2. 用最短的 Cloze 片段测试一次识别效果，确认正面、背面和上下文都符合预期。
3. 如果你需要同级或新级的多空结构，再引入快捷命令，而不是手动在大段文本里频繁改语法。
4. 当你发现 Cloze 泄露太多或太少上下文时，回到设置页调上下文模式和显示其他 Cloze 的开关。

## 相关设置 / 相关命令
- 相关设置：高亮转 Cloze、粗体转 Cloze、花括号转 Cloze、Cloze 上下文模式、显示其他 Cloze。
- 参考页面： [高级 Cloze](./advanced-cloze.md)、[卡片与笔记设置](../settings/flashcards-and-notes.md)。

## 常见错误
- 同时启用多种来源，却没有统一规则，最后一篇笔记里混了好几套风格。
- Cloze 片段过长或过短，导致真正考察点不明确。
- 用快捷命令前没有理解同级和新级的差别。

## FAQ
- **同级和新级有什么区别**：可以把它理解成“继续往当前那组空里补内容”和“开启一组新的 Cloze 层级”。
- **为什么我看到其他空也露出来了**：这通常和“显示其他 Cloze”的设置以及当前上下文模式有关。
- **高亮、粗体、花括号必须全开吗**：不必须。建议先选最贴近你写作习惯的一种，再慢慢扩展。

## 排错与风险提示
- 如果你在同一库里用太多 Cloze 风格，会让协作和后期维护变得困难。
- 如果你追求复杂多空结构，最好先在少量测试笔记里验证再推广。
