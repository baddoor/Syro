# 复习队列侧边栏

![复习队列侧边栏](../../assets/media/en/note-review-queue.jpg)

> 提示：当前仓库可复用的截图多来自较早的英文界面，但布局和入口位置仍可作为对照。

## 这是什么
- 复习队列侧边栏是笔记复习工作流的主控台。它把候选笔记按到期状态、标签和当前上下文组织起来。
- 如果你只会从命令面板打开一篇笔记，而不会使用队列侧边栏，很多效率提升点都会被错过。

## 从哪里进入
- 命令 `Open Notes Review Queue in sidebar`。
- 启动时自动打开侧边栏的设置。
- 在打开笔记复习后，通过右侧工作区长期固定该视图。

## 适合什么场景
- 你要在多篇候选笔记中决定今天先读哪篇。
- 你想按新笔记、逾期、今天、明天和未来分区来做轻重缓急。
- 你想一边看队列一边在主编辑区打开原文。

## 具体步骤
1. 打开侧边栏视图，先观察它如何按新笔记、逾期、今天、明天和未来分区。
2. 单击某个条目打开原文，感受它作为阅读入口而不是卡片入口的定位。
3. 用右键菜单测试“新标签页打开”“向右分屏”“弹出窗口”等操作，确认哪个最适合你的工作区布局。
4. 如果队列看起来太长，继续学习标签筛选和忽略规则，而不是立刻怀疑数据损坏。

## 相关设置 / 相关命令

- 相关设置：启动时自动打开侧边栏、隐藏过滤条、显示滚动百分比、自动展开时间线。

## 常见错误
- 把侧边栏当成“静态列表”，没有使用它的筛选、排序和上下文菜单。
- 看到队列为空就立刻判断插件坏了，而没有先确认笔记是否被追踪或同步是否完成。
- 在笔记复习场景里寻找闪卡的 Again/Hard/Good/Easy 按钮。

## FAQ
- **为什么侧边栏里有分区**：因为它需要把“现在就该看”和“未来再看”的笔记区分开，避免所有笔记挤在一个平面列表里。
- **我能在侧边栏里直接管理标签和优先级吗**：可以，标签筛选和优先级编辑就是为了减少你离开侧边栏的次数。
- **侧边栏是不是只能配合笔记复习用**：主要是，但你仍然可以把它当成工作区里的上下文导航工具。

## 排错与风险提示
- 如果你在多窗口或分屏环境里工作，注意你评分和你查看的窗口是否还是同一个对象，避免误以为“打开了别的笔记”。
- 如果条目状态和你的直觉明显不符，先回到 [数据与同步总览](../data-and-sync/index.md) 检查同步和缓存。
