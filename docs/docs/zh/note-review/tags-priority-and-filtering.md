# 标签、优先级与筛选

> 待补截图：建议补一张标签筛选栏截图，以及一张在侧边栏内修改优先级的截图。

## 这是什么
- 这页聚焦队列管理：如何用标签缩小范围、如何忽略标签、如何排序，以及如何通过优先级让重要笔记更早出现。
- 它不是算法页，而是“怎么把工作台整理成你愿意每天打开的样子”的页面。

## 从哪里进入
- 笔记复习侧边栏顶部的筛选和排序区域。
- 笔记条目上的优先级编辑入口。
- 与标签拖拽、忽略和 frontmatter 写入相关的交互。

## 适合什么场景
- 你有太多待复习笔记，想先处理某个主题或课程。
- 你想把琐碎标签隐藏掉，让队列更安静。
- 你希望某些长期重要但不一定最近刚到期的笔记更常进入视野。

## 具体步骤
1. 先在侧边栏观察当前标签列表和排序方式，确认你的仓库里是否已经形成可用的标签体系。
2. 用忽略标签功能去掉干扰项，让队列先回到可读状态。
3. 在真实工作中逐步补优先级：先给最重要的少数笔记设高优先级，而不是一次性批量设置所有笔记。
4. 如果你依赖标签拖拽写回元数据，先用一两篇测试笔记验证你的 frontmatter 习惯，再批量操作。

## 相关设置 / 相关命令
- 相关设置：忽略标签、隐藏过滤条。
- 相关页面： [复习队列侧边栏](./review-queue-sidebar.md)、[Timeline](./timeline.md)。

## 常见错误
- 把所有标签都保留下来，导致真正有用的筛选项埋在噪音里。
- 优先级设得过满，最后所有内容都“同样重要”。
- 没有先确认 frontmatter 写法，就直接大量拖拽标签到笔记。

## FAQ
- **优先级和到期状态谁更重要**：它们解决不同问题：到期状态决定时间维度，优先级决定同一时间层里的先后顺序。
- **忽略标签会删除笔记上的标签吗**：忽略更偏向视图和队列层面的过滤，不等于永久删除原始标签。
- **如果我不用标签，还需要这一页吗**：仍然建议阅读，因为优先级和过滤条显示方式会影响你长期使用侧边栏的效率。

## 排错与风险提示
- 如果多人共享同一仓库，请先约定标签用途；否则一个人为筛选而改动的标签，可能影响另一个人的写作习惯。
- 如果优先级行为与你想象不一致，先确认当前算法和队列不是由其他设置共同影响。
