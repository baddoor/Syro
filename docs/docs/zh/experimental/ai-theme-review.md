# AI 主题复习

> 待补截图：建议补一张 AI 标签页、一张 AI 主题包创建面板、一张 Native / AI 切换后的牌组树截图。

## 这是什么
- AI 主题复习把已有笔记和卡片按照主题包重新组织，适合你在大库里围绕一个主题快速收拢相关内容。
- 它当前依赖额外的检索能力；文档默认把它当作实验与高级能力，而不是普通新手功能。

## 从哪里进入
- 设置页中的 AI 标签。
- 闪卡工作区里与 AI 牌组或主题包相关的入口。
- 与 Smart Connections 和可选 LLM 精排有关的配置项。

## 适合什么场景
- 你已经有大量分散的笔记和卡片，想围绕一个主题临时重组复习入口。
- 你愿意处理额外依赖，换取更强的主题召回能力。
- 你需要理解“为什么 AI 主题包入口可见，但当前保存或重生成失败”。

## 具体步骤
1. 先确认当前环境是否检测到可用的检索器；没有检索器时，这一能力不会按完整路径工作。
2. 再决定是否启用 LLM 精排。如果你只想先确认主题包流程，建议先在不启用 LLM 的情况下验证基础召回。
3. 从少量主题包开始测试，观察主题提示词、最终条目数和排序方式是否符合预期。
4. 如果创建或保存失败，优先检查依赖、模型设置和当前检索器状态，而不是直接怀疑牌组树或主数据。

## 相关设置 / 相关命令
- 相关设置：检索器、主题包默认条目数、排序方式、是否启用 LLM 精排、Provider、Model、System Prompt、严格 JSON 输出。
- 相关页面： [License 与支持者能力](./license-and-supporter.md)、[数据文件参考](../appendix/data-files-reference.md)。

## 常见错误
- 没有准备检索依赖就直接进入 AI 主题包流程。
- 一上来就同时调主题提示词、排序、LLM 和结果数量，最后无法判断是哪一步出了问题。
- 把 AI 主题复习当成主数据结构的替代品，而不是一层附加能力。

## FAQ
- **没有 Smart Connections 能用吗**：基础文档口径是：没有可用检索器时，这条工作流不应该被当作稳定能力来依赖。
- **LLM 精排是不是必须开启**：不是。先把基础召回跑通，再决定是否引入 LLM。
- **AI 主题包会不会改坏我的原始笔记**：它更像额外生成的复习入口和附加数据层，不是直接改写原文。

## 排错与风险提示
- AI 主题包会引入额外数据文件与额外依赖，备份时不要只看主卡片数据。
- 任何与外部检索或模型有关的能力，都可能比稳定主线更容易受到环境变化影响。
