# 实验与高级总览

## 这是什么
- 这一组页面收纳那些不适合放进稳定主线教程的内容：AI 主题复习、License / 支持者能力、隐藏入口、调试开关、兼容逻辑与高级排障口径。
- 它们并不是“不能用”，而是通常需要更多前置条件、更多解释，或更强的风险意识。

## 从哪里进入
- AI 标签页、License 标签页、部分支持者开关、调试和兼容字段相关入口。
- 某些入口在界面里可见，但文档会明确说明它们当前属于实验、高级或兼容范畴。

## 适合什么场景
- 你想启用 AI 主题复习或了解其依赖。
- 你正在处理支持者能力、License 校验或高级 Cloze 能力。
- 你在排障时需要查看隐藏、调试或兼容相关说明。

## 具体步骤
1. 先看 [AI 主题复习](./ai-theme-review.md)，确认你的仓库、Smart Connections 和模型配置是否满足条件。
2. 如果你关心支持者能力和授权边界，再读 [License 与支持者能力](./license-and-supporter.md)。
3. 当你需要解释为什么某个开关存在但不建议普通用户启用时，继续看 [隐藏、调试与兼容](./advanced-debug-compat.md)。

## 相关设置 / 相关命令
- 稳定设置请回到 [设置总览](../settings/index.md)。
- 数据文件与迁移说明请回到 [数据与同步总览](../data-and-sync/index.md)。

## 常见错误
- 把实验与高级区当成新手第一站。
- 看到开关就默认它已经完全可用，而不先看依赖和限制。
- 在排障时长期保留高风险或高噪音设置。

## FAQ
- **为什么这些内容不放进主教程**：因为它们需要更强的前置条件、更多限制说明，或者仍然带有兼容 / 实验色彩。
- **实验区是不是就不可靠**：不完全是。它们可能已经可用，但不适合作为普通用户默认路径。
- **我只关心稳定工作流，需要看这里吗**：只有当你触碰 AI、支持者能力、调试或兼容问题时，才需要回来查。

## 排错与风险提示
- 实验与高级功能最容易和旧截图、旧教程、旧字段混在一起；遇到冲突时以当前实现和本文档口径为准。
- 开启实验能力前先备份，尤其是当它会影响额外数据文件或外部依赖时。
