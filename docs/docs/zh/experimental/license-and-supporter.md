# License 与支持者能力

## 这是什么
- 这一页负责解释哪些功能会因为支持者能力或授权状态而表现不同，以及你应该如何理解这些边界。
- 文档不会把支持者能力写成营销文案，而是只描述可见开关、可用边界和排障思路。

## 从哪里进入
- 设置页中的 License 标签。
- 高级 Cloze、Anki Cloze、代码块 Cloze、LaTeX Cloze 等与支持者能力相关的可见开关。

## 适合什么场景
- 你想知道为什么某个设置明明看得见却不能正常启用。
- 你在比较免费路径和支持者路径的差别。
- 你需要在排障时先排除授权状态相关的问题。

## 具体步骤
1. 先确认当前能力属于哪一类：完全稳定公开能力、支持者增强能力、还是实验能力叠加支持者条件。
2. 查看 License 标签页是否已经提供明确的当前状态反馈。
3. 当某个高级能力无法启用时，先判断是授权边界、依赖缺失，还是内容写法本身有问题。
4. 如果你只是想完成主工作流，优先回到稳定主线，不必为了单个高级特性阻塞整个上手过程。

## 常见错误
- 把“按钮可见”误判为“功能一定已可用”。
- 授权失败时只怪内容写法，不去先排除 License 状态。
- 把支持者能力和实验能力的边界混在一起。

## FAQ
- **支持者能力会影响稳定主线吗**：主线仍然可用，但某些增强场景会因为授权状态而表现不同。
- **我不打算用支持者能力，还需要看这里吗**：通常不需要；只有当你遇到相关开关或说明不一致时，再回来查。
- **授权失败是不是就会损坏原数据**：通常不会。它更多影响某些能力是否开放，而不是直接破坏已有数据。

## 排错与风险提示
- 当同一项能力同时依赖授权和额外环境条件时，排障路径会更复杂，最好一次只排一个变量。
- 如果你在团队环境里写教程，请把“支持者增强”明确标出来，避免其他人以为这是默认能力。
