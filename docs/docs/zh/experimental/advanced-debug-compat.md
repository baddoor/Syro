# 隐藏、调试与兼容

## 这是什么
- 这页集中解释那些不适合放入主教程的调试开关、隐藏入口、兼容字段和排障向能力。
- 如果某个功能当前缺少稳定公开入口，文档会在这里保守描述，而不是把它包装成正式主线能力。

## 从哪里进入
- UI 标签页中的调试相关开关。
- 代码中仍保留、但 UI 中不一定完整暴露的兼容字段。
- 某些只在排障或迁移时才会提到的命令和状态。

## 适合什么场景
- 你在排障，需要短期开启更多输出。
- 你在查看旧教程或旧 JSON，遇到今天主教程里看不到的字段。
- 你想知道为什么某个内部模态框或隐藏能力不在主导航里。

## 具体步骤
1. 先确认你是不是在做排障；如果只是日常使用，大多数情况下不需要碰这里。
2. 短期启用调试输出时，同时记下你改了什么，问题确认后尽快恢复默认。
3. 遇到兼容字段时，不要急着手改；先确认它是不是历史迁移产物，或者只是内部保留状态。
4. 对于没有稳定公开入口的能力，文档只把它们视为附录级或实验级信息，而不是主导航正式功能。

## 常见错误
- 为了排障长期保留高噪音调试开关。
- 把兼容字段当成新功能推荐给其他用户。
- 根据内部名称推断用户可见行为，而不对照当前界面。

## FAQ
- **为什么有些名字我能在代码或旧截图里看到，但主教程里没有**：因为它们缺少稳定公开入口，或更适合放在兼容 / 调试说明中。
- **调试开关能不能一直开**：技术上也许可以，但对大多数用户来说会增加噪音和误判成本。
- **兼容字段什么时候才需要看**：通常是在迁移、恢复旧库或做深度排障时。

## 排错与风险提示
- 这里的信息最容易和旧版本经验混杂，操作前一定先做备份。
- 排障时如果同时改了太多隐藏开关，问题短期可能看起来好了，但你很难知道真正原因是什么。
