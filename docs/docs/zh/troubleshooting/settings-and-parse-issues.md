# 设置与解析问题

## 这是什么
- 这页专门处理内容识别、设置生效和显示结果相关的异常，例如没出卡、上下文不对、开关没效果或支持者能力看起来失灵。
- 如果症状主要发生在单篇内容、单类设置或单个工作流里，优先从这里排查。

## 从哪里进入
- 卡片解析、Cloze、分隔符、上下文模式、Notes / Flashcards 设置、支持者相关开关。

## 适合什么场景
- 你写了问答或 Cloze，但牌组树里没看到卡片。
- 你改了上下文设置、队列设置或算法 UI 控件，但体感变化不符合预期。
- 你不确定问题来自内容本身、设置本身还是授权边界。

## 具体步骤
1. 先用最小样例重现问题，避免把大段复杂笔记带来的干扰当成设置问题。
2. 检查相关设置是否真的属于稳定生效项；尤其在算法和高级 Cloze 场景里，不要默认所有控件都已完整接线。
3. 如果是支持者能力相关，先排除授权或依赖边界。
4. 如果内容样例和设置都看起来没问题，再回到 [数据同步与迁移问题](./data-sync-and-migration-issues.md) 判断是不是状态层异常。

## 相关设置 / 相关命令
- 相关页面： [卡片与笔记设置](../settings/flashcards-and-notes.md)、[算法与 WMS](../settings/algorithms-and-wms.md)、[高级 Cloze](../card-authoring/advanced-cloze.md)。

## 常见错误
- 用大而复杂的真实笔记直接排查，导致无法隔离问题。
- 把模拟器 / 预留项当成稳定生效参数。
- 忽略支持者能力或实验能力的边界，只看见一个开关就认定它应该生效。

## FAQ
- **为什么我改了 WMS 某个控件却感觉几乎没变化**：先确认它是不是当前真实生效参数；有些控件更像模拟器或预留项。
- **为什么有时 Cloze 正常，有时又不正常**：往往是内容写法、上下文设置和支持者相关能力共同作用，而不是单一按钮导致。
- **没出卡一定是同步问题吗**：不一定。更多时候先看写法和解析设置。

## 排错与风险提示
- 排查解析问题时最容易把多个变量一起改掉，导致最后谁也说不清真正原因。
- 如果你要把异常样例发给别人看，注意先脱敏并保留最小复现。
