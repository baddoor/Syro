# 数据同步与迁移问题

## 这是什么
- 这页处理那些更像“状态层”异常的问题：文件里有数据但界面没显示、升级后文件结构变了、Overlay 看起来不理解、旧存储模式和当前口径不一致等。
- 只要问题超出单篇内容写法，就很可能需要看这里。

## 从哪里进入
- 同步命令、缓存重建、数据文件、迁移逻辑、历史存储说明。

## 适合什么场景
- 你明明知道自己有数据，却觉得界面和文件对不上。
- 你升级后看到新文件或旧字段，不知道是不是异常。
- 你需要决定是等同步完成、重建缓存，还是进一步检查数据文件。

## 具体步骤
1. 先确认问题是否真的属于状态层：换一份最小内容样例仍然异常、重开工作区仍异常，才继续往下。
2. 检查自动增量同步、缓存持久化和当前最近一次大改动之间的关系。
3. 对照 [存储模型](../data-and-sync/storage-model.md)，确认你看到的文件是不是本来就应该存在。
4. 如果牵涉到旧教程或旧版本数据，再进入迁移说明，而不是直接把旧路径抄到当前版本。

## 常见错误
- 一看到多个 JSON 文件就误判为损坏。
- 不做备份就删除 Overlay 或独立数据文件。
- 把旧存储模式误当成当前正式推荐方案。

## FAQ
- **为什么升级后数据文件变多了**：因为不同类型的数据被拆分管理了，这通常是设计结果，不一定是异常。
- **什么时候该重建缓存**：当你已经判断属于状态层异常，并且常规同步没有恢复时，才把它提到更前。
- **只备份一个文件够吗**：通常不够。至少要按当前文档列出的完整数据组来备份。

## 排错与风险提示
- 手工数据操作前请务必完整备份。
- 如果你同时跨版本迁移、改设置、改文本，问题可能不是单点引起的，排查顺序要更保守。
