# Repo Commit Workflow

- For any task in this repository that edits code or local project files, use the `auto-local-commit-zh` skill by default.
- After completing the task, create a local git commit without waiting for a separate commit request.
- Stage only the files for the current task and leave unrelated user changes out of the commit.
- Use a Simplified Chinese commit message that follows the `auto-local-commit-zh` skill format.
- If a task makes no relevant git-tracked changes, state that no commit was created.
